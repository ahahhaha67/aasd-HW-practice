Неявный интерфейс шаблона unique
Неявный интерфейс — это набор требований к типу T, которые нигде не объявлены явно, но необходимы для корректной компиляции шаблона. Компилятор проверяет их только в момент инстанцирования.
Для unique<T> тип T должен поддерживать operator== для сравнения элементов, operator= для записи уникального элемента и конструктор копирования при копировании элементов внутри шаблона.
Если тип T не удовлетворяет этим требованиям — ошибка компиляции возникает только при попытке использовать шаблон с конкретным типом.

Недостатки явного интерфейса
Избыточная связанность — тип T обязан наследоваться от базового класса или реализовывать конкретный интерфейс, даже если у него уже есть нужная функциональность под другим именем.
Нет утиной типизации — встроенные типы (int, double, char) не могут реализовать интерфейс, так как не поддерживают наследование → пришлось бы писать обёртки.
Жёсткость — нельзя использовать сторонние типы без их модификации, что нарушает принцип открытости/закрытости.
Лишние зависимости — типу T нужно знать об интерфейсе шаблона, хотя логически они независимы.
Снижение повторного использования — шаблон с неявным интерфейсом работает с любым типом у которого есть ==, тогда как явный интерфейс жёстко ограничивает круг совместимых типов.
